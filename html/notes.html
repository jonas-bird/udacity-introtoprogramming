Lesson 1: Styling with CSS

When the browser renders the DOM the block level elements are rendered as boxes.

Each box has:

  1. Padding between the content of the element and the border
  2. A border (by default not visible, but it can be made wider and displayed)
  3. A margin that controls how close other elements are displayed. Negative values can be used to create overlap

Fonts

Rather than have each font related property/value pair seperated, they can be combined with a shorthand expression.

font: italic 24px Times;

is the same as


                     font-style: italic;
                     font-size: 24px;
                     font-family: Times;

The declarations need to be in the order font: font-style font-variant font-weight font-size font-family; Both font-size and font-family are required, the other properties are optional and have default values

For more info Css-tricks


It is possible to instruct a browser to download a specific font from a repository such as Google fonts This has made the concept of 'web safe' fonts less of a concearn and opened up a wider range of design possibilities. Using an online source for non-standard fonts does not remove the need for a font stack with generic options as some users will have their browsers configured (to conserve bandwidth, as part of a styling plugin, or for privacy reasons) configured to not download 3'rd party assets like fonts

To make use of Googl's fonts use the following snippet. Change the name of the font in the URL parameter as well as in the CSS. 


            <html>
  <head>
    <link rel="stylesheet"
          href="https://fonts.googleapis.com/css?family=Tangerine">
    <style>
      body {
        font-family: 'Tangerine', serif;
        font-size: 48px;
      }
    </style>
  </head>
  <body>
    <div>Making the Web Beautiful!</div>
  </body>
</html>

In order to group elements together, and to apply changes in positioning to the assembladge of elements, rather than make changes to the individual elements, we can make use of placing a div around other block level elements, including other div elements. HTML was originally designed to handle the layout for basic text documents. As the types of information represented, and the design and layout for the content have become more complex, CSS has adapted to allow for more fine-tuned placement of elements, and more complex displays. One of the common tools in current use is the flexible box or 'flexbox' model. Using a flexbox layout changes the way that the elements behave in response to differing amounts of screen real-estate with the flex-wrap property

Setting flex-wrap: wrap; in the containing block level element causes the inner elements to move to a new-line rather than compress
